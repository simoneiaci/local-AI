🚀 Setup Guide

Get your local AI stack running in about an hour.

1

Install Ollama

The primary LLM runtime. Exposes an OpenAI-compatible API.

brew install ollama
brew services start ollama
2

Pull your first models

Start with the essentials — a daily driver and a coding model.

# Daily driver (~7 GB)
ollama pull gemma3:12b

# Coding specialist (~14 GB)
ollama pull devstral

# Quick chat (~3 GB)
ollama pull phi4-mini

# Embeddings for RAG (~0.3 GB)
ollama pull nomic-embed-text
3

Configure shell aliases

Add to ~/.zshrc for quick model switching:

# Ollama config
export OLLAMA_KEEP_ALIVE=5m
export OLLAMA_MAX_LOADED_MODELS=1
export OLLAMA_NUM_GPU=99
export OLLAMA_HOST=0.0.0.0:11434

# Quick model switching
alias ai-chat="ollama run phi4-mini"
alias ai-general="ollama run gemma3:12b"
alias ai-code="ollama run devstral"
alias ai-reason="ollama run phi4-reasoning"
alias ai-power="ollama run mistral-small3.1:24b"
alias ai-status="ollama ps"
4

Install Open WebUI

A ChatGPT-like interface for all your local models.

# Start Podman machine (macOS requirement)
podman machine start

podman run -d -p 3000:8080 \
  -e OLLAMA_BASE_URL=http://host.containers.internal:11434 \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart=always \
  ghcr.io/open-webui/open-webui:main

Open http://localhost:3000

5

Test it!

Chat with your first local model:

ollama run gemma3:12b "Explain quantum computing in simple terms"

Workflow Pattern: Plan → Build

Community consensus (r/opencodeCLI) is to split coding agent work into two phases. Even with the best proprietary models, a single call trying to reason deeply and emit correct code tends to underperform a two-step split. On a 24 GB M4 Pro with approved local models:

PhaseModelWhy
Plan / thinkphi4-reasoning75.3% AIME — strong step-by-step reasoning
Build / codedevstral53.6% SWE-Bench — best approved coder
Debug / reviewmistral-small3.1:24b128K context, good at tracing bugs
# Plan the approach with a reasoning model
OPENCODE_MODEL=phi4-reasoning opencode "Design a retry layer"

# Switch to devstral to implement
ai-use-coding   # alias sets OPENCODE_MODEL=devstral
opencode "Now implement the plan"

Context Budget — Keep Under ~100K Tokens

Even models advertising 128K context degrade noticeably after ~100K (hallucinations, lost references, language drift). Recommended working budgets:

ModelAdvertisedWorking budget
gemma3:12b128K≤ 32K (sweet spot)
mistral-small3.1:24b128K≤ 100K
granite3.3:8b128K≤ 100K
devstral32K≤ 32K
phi4-reasoning16K≤ 16K (hard limit)

Rules of thumb: summarize-and-restart at ~20K increments, reserve ~20% of context for the output, and feed directory listings first before pulling in specific files.

📱 Remote Access

Access your local AI from your iPhone — anywhere in the world.

Three Options

Option A

Tailscale (Simplest)

Private mesh VPN. End-to-end WireGuard encryption. No port forwarding, no config. Install on Mac + iPhone, done.

Recommended5 min setupE2E encrypted
# Install
# Download from tailscale.com (standalone .pkg)
# Install Tailscale on iPhone from App Store
# Sign in with same account on both

# Set Ollama to listen on all interfaces
export OLLAMA_HOST=0.0.0.0:11434

# Access from phone:
# http://100.x.y.z:3000 (Open WebUI)
Option B

Cloudflare Tunnel (Zero-Trust)

No port forwarding needed. Outbound tunnel from your Mac to Cloudflare. Free tier. Automatic HTTPS.

15 min setupZero-trustFree
brew install cloudflare/warp/cloudflared
cloudflared tunnel login
cloudflared tunnel create my-ai-tunnel

# ~/.cloudflared/config.yml
# tunnel: my-ai-tunnel
# ingress:
#   - hostname: myai.example.com
#     service: http://localhost:3000

cloudflared tunnel run my-ai-tunnel
Option C

Caddy + DDNS (Public IP)

Use your home's public IP. Caddy handles automatic HTTPS via Let's Encrypt. Requires router port forwarding.

20 min setupFull controlPublic IP
brew install caddy

# Caddyfile:
# myai.duckdns.org {
#     reverse_proxy localhost:3000
# }

caddy run --config Caddyfile

# Forward port 443 on your router
# Set up DuckDNS or Cloudflare DDNS

PWA on iPhone

Make it feel like a native app Open your AI URL in Safari on iPhone → tap Share → "Add to Home Screen". Open WebUI runs as a Progressive Web App (PWA) — fullscreen, no browser toolbar. Your own ChatGPT icon on your home screen.
Keep your Mac awake Run caffeinate -s (when plugged in) or set System Settings → Displays → Advanced → "Prevent automatic sleeping when the display is off". Tailscale cannot wake a sleeping Mac.

Memory Management

# Auto-unload models after 5 min idle
export OLLAMA_KEEP_ALIVE=5m

# Only 1 model at a time (critical for 24 GB!)
export OLLAMA_MAX_LOADED_MODELS=1

# Force unload all models right now
for model in $(ollama ps | tail -n +2 | awk '{print $1}'); do
  curl -s http://localhost:11434/api/generate -d "{\"model\":\"$model\",\"keep_alive\":0}" > /dev/null
done
