⚙️ Setup

Five scripts take you from zero to a fully running local AI stack. Run them in order.

1

Phase 1 — Ollama + Core Models

Installs Ollama, pulls the base model stack, sets shell aliases and environment variables.

bash scripts/phase1-setup.sh
2

Phase 2 — Coding Tools

Configures Continue.dev for VS Code, installs OpenCode and Aider, pulls devstral and smollm2:1.7b for autocomplete.

bash scripts/phase2-coding-tools.sh
3

Phase 3 — Open WebUI

Starts a Podman container with the full Open WebUI interface connected to your local Ollama.

bash scripts/phase3-webui.sh
# Then open: http://localhost:3000
4

Phase 4 — Dashboard

Starts the metrics exporter (host process) and a Podman container serving the live system dashboard.

bash scripts/phase4-dashboard.sh
# Then open: http://localhost:9090
5

Phase 5 — Remote Access

Choose Tailscale (private VPN), Caddy + DuckDNS (public HTTPS), Cloudflare Tunnel, or all of them.

bash scripts/phase5-remote.sh
# Choose: 1=Tailscale  2=Caddy  3=Cloudflare  4=Both
After Phase 1, source your ~/.zshrc once, then use ai-stack-start / ai-stack-stop to run everything with one command.

🤖 Model Stack

All models run locally via Ollama. Only one 14 GB model fits in RAM at a time — swap as needed.

⚠️ With 24 GB unified memory, about 14–16 GB is available for models. smollm2:1.7b can stay loaded alongside any other model.

Plan → Build Workflow

For complex tasks, split across two models to get better results:

1
Plan phi4-reasoning
Analyse the problem, identify edge cases, design the approach. Ask it to think step-by-step before writing any code.
→
2
Build devstral
Feed the plan to devstral. It writes clean, working code fast — no need for it to reason from scratch.
💡 Keep context under ~100K tokens even for 128K-context models. Performance drops near the limit on Apple Silicon due to memory bandwidth constraints.

Architecture

Two processes work together:

ComponentWhereDoes
metrics-exporter.pyMac hostWrites /tmp/ai-metrics.json every 3s. Also runs control server on port 9091.
local-ai-dashboardPodman containerServes the web UI on port 9090. Reads metrics via bind-mount, calls Ollama API directly.

Control API

The dashboard's start/stop buttons call the host control server:

POST http://localhost:9091/control
{"action": "ollama_stop"}    # or: ollama_start
{"action": "webui_stop"}     # or: webui_start
{"action": "podman_stop"}    # or: podman_start
{"action": "tailscale_down"} # or: tailscale_up

Useful Commands

podman logs local-ai-dashboard       # container logs
podman stop local-ai-dashboard        # stop dashboard
pkill -f metrics-exporter.py         # stop host exporter
cat /tmp/ai-metrics-exporter.log     # exporter debug log

📱 Remote Access

Access Open WebUI from your iPhone or any device, from anywhere in the world.

Method Comparison

MethodSecuritySetupBest for
TailscaleEnd-to-end E2EE5 minPersonal use — simplest & safest. Recommended.
Cloudflare TunnelZero-trust15 minNo port forwarding, free, needs a domain
Caddy + DuckDNSHTTPS/TLS20 minFull control, public URL, requires port forwarding

Option 1: Tailscale (Recommended)

1

Install Tailscale on Mac

Download the standalone .pkg (not App Store) from tailscale.com/download/mac and sign in.

2

Connect

tailscale up
3

Install Tailscale on iPhone

App Store → Tailscale → sign in with the same account. Your Mac now has a stable private IP visible in the dashboard.

4

Access from iPhone

Open Safari → http://<tailscale-ip>:3000 → Share → Add to Home Screen for a native-looking PWA.

⚠️ Keep your Mac awake while using it remotely: caffeinate -s & (plugged in only), or System Settings → Displays → Advanced → Prevent automatic sleeping.

Option 2: Cloudflare Tunnel

brew install cloudflare/warp/cloudflared
cloudflared tunnel login
cloudflared tunnel create my-ai-tunnel
# Configure ~/.cloudflared/config.yml, then:
cloudflared tunnel run my-ai-tunnel

Run the setup script

bash scripts/phase5-remote.sh
# 1 = Tailscale only
# 2 = Caddy + DuckDNS only
# 3 = Cloudflare Tunnel only
# 4 = Tailscale + Caddy (both)
