02 / Setup

Six phases, fully automated.

Six scripts take you from zero to a fully running local AI stack. Run them in order from the repo root. Phase 6 layers community-recommended improvements on top.

1

Phase 1 — Ollama + Core Models

Installs Ollama, pulls the base model stack, configures shell aliases and environment variables in ~/.zshrc.

bash scripts/phase1-setup.sh
2

Phase 2 — Coding Tools

Configures Continue.dev for VS Code, installs OpenCode and Aider, pulls gemma3:12b + granite3.3:8b for coding and smollm2:1.7b for tab autocomplete.

bash scripts/phase2-coding-tools.sh
3

Phase 3 — Open WebUI + Pipelines

Starts Podman containers for Open WebUI (port 3000) and Pipelines middleware (port 9099).

bash scripts/phase3-webui.sh
# Then open: http://localhost:3000
4

Phase 4 — Dashboard

Starts the metrics exporter (host process) and a Podman container serving the live system dashboard at port 9090.

bash scripts/phase4-dashboard.sh
# Then open: http://localhost:9090
5

Phase 5 — Remote Access

Choose Tailscale (private VPN), Caddy + DuckDNS (public HTTPS), Cloudflare Tunnel, or a combination.

bash scripts/phase5-remote.sh
# 1=Tailscale  2=Caddy  3=Cloudflare  4=Both
6

Phase 6 — Community Improvements (optional)

Layers practitioner-recommended upgrades: LM Studio (MLX backend, ~20–30% faster on Apple Silicon — community-reported), mlx-lm CLI, web-search MCP servers (Tavily / Brave / fetch) with a shared local config and Continue.dev integration, the lightweight pi coding agent, and TurboQuant model variants. Includes a native macOS menu bar app.

bash scripts/phase6-improvements.sh
# Idempotent — safe to re-run. Each step prompts (y/N).
After Phase 1, source your ~/.zshrc once — then use ai-stack-start and ai-stack-stop to manage the whole stack with a single command.
03 / Models

Right model for every task.

Models run locally through LM Studio by default, with Ollama available as an alternate runtime. Only one large model fits in RAM at a time — smollm2:1.7b and nomic-embed-text stay loaded alongside any other.

⚠️ With 24 GB unified memory, ~14–16 GB is available for models after macOS overhead. Swap models as needed — unloading takes a second.

Control API

The dashboard proxies start/stop buttons to the host control server on port 9091. Requests without CONTROL_TOKEN are rejected:

POST http://localhost:9091/control
Authorization: Bearer <CONTROL_TOKEN>

{"action": "ollama_stop"}     # or: ollama_start
{"action": "lmstudio_stop"}   # or: lmstudio_start
{"action": "webui_stop"}      # or: webui_start
{"action": "pipelines_stop"}  # or: pipelines_start
{"action": "podman_stop"}     # or: podman_start
{"action": "tailscale_down"}  # or: tailscale_up

06 / Web Search

Live internet access, fully local.

Give your local models live internet access. Search results are fetched and summarised privately, right inside Open WebUI — nothing leaves your Mac.

✓ Web search is active. Engine: SearXNG — self-hosted on port 8080, zero external requests, unlimited searches.

How It Works

Open WebUI has built-in web search. When enabled, a 🌐 button appears in every chat. Toggle it before sending a message — the model searches the web, reads the top results, and cites its sources, all without leaving your Mac.

Using Search in Open WebUI

1

Open a chat

Go to localhost:3000 and open any conversation.

2

Enable web search

Click the 🌐 icon in the message bar before sending your prompt.

3

Ask your question

The model searches SearXNG, reads the top results, and cites sources inline. All traffic stays on your Mac.

07 / Open WebUI

Beyond basic chat.

Open WebUI has powerful features built in. All settings live in Admin Panel — click your avatar → Admin Panel → Settings.

✓ Pipelines container is running at localhost:9099 — started automatically by ai-stack-start.

1 · RAG / Knowledge Base

Upload documents — PDFs, Word files, receipts, Swift docs — and let the model answer questions from them using semantic search. Far more accurate than keyword matching.

1

Set the embedding model

Admin Panel → Settings → Documents. Set Embedding Model to nomic-embed-text (already pulled). Save.

2

Create a Knowledge collection

Workspace → Knowledge → New Knowledge. Name it (e.g. "Tax Docs 2025"). Upload PDFs — 730 forms, CU, F24, ricevute.

3

Use in chat

Click + in a new chat → select your Knowledge collection. Or use 📎 to attach a single file directly. Then ask: "Quali spese mediche posso detrarre nel 730/2025?"

💡 Best local model for Italian tax RAG: granite3.3:8b for multilingual 128K-context document Q&A. Use phi4-reasoning when the task is mostly reasoning rather than retrieval.

2 · Memory

Open WebUI remembers facts across conversations — your name, that you do iOS dev, your tax situation — building a personal context that persists between sessions.

1

Enable Memory

Admin Panel → Settings → Personalization. Toggle Memory on. Save.

2

Seed your memory (optional)

Profile → Memory. Add facts manually: "I'm an iOS developer using Swift 6 and TCA" or "I live in Italy and file 730/redditi with a partita IVA".

3

Let it build automatically

During conversations the model extracts and saves relevant facts. Review and edit any time in Profile → Memory.

3 · Code Execution (Artifacts)

Models can write Python or JavaScript and run it inline, showing output. Ideal for tax calculations, data processing, or verifying code snippets.

1

Enable Code Interpreter

Admin Panel → Settings → Interface. Toggle Code Interpreter (or "Artifacts") on. Save.

2

Ask the model to run code

Ask naturally: "Calculate my IRPEF for €45,000 gross income" or "Write and run a Python script to parse this CSV". A Run button appears — tap to execute and see output inline.

⚠️ Code runs inside the Open WebUI Podman container — sandboxed and safe.

4 · Pipelines

Custom Python middleware that intercepts every message. Use it to auto-translate Italian documents, post-process model output, add rate limiting, or inject context automatically.

1

Verify the Pipelines container

podman ps | grep pipelines
# Should show open-webui-pipelines on port 9099
2

Connection settings

Admin Panel → Settings → Connections → OpenAI API → add:

FieldValue
URLhttp://host.containers.internal:9099
API Key0p3n-w3bu!

Click Verify — green checkmark confirms it's connected.

3

Install a pipeline

Admin Panel → Pipelines → Install from GitHub URL or upload a .py file. Browse examples at github.com/open-webui/pipelines.

5 · Image Input (Vision)

Already works — select gemma3:12b and attach an image. Useful for reading receipts, Xcode error screenshots, diagrams, or scanned documents.

Access custom models

2
